#How to Use Backblaze B2 as External Storage in Web Applications and Cloud Services
In this example, we used Nextcloud

##1. Create the Subdomain
Go to the Domains option in the SiteGround menu and create a subdomain.
 Note: Only an administrator can perform this action.
 Enter just the desired name (e.g., nextcloud); SiteGround will automatically complete the address.

![Creaci贸n de subdominio en SiteGround](https://i.ibb.co/JRxW4MqG/one.webp)  

##2. Access the File Manager
Go to the Site > File Manager menu.

Locate the folder of the subdomain you created (it will contain a public_html subfolder).

![Acceso al File Manager](https://i.ibb.co/mVJkT7xC/two.webp)  

##3. Download and Upload Nextcloud
Download the Nextcloud .zip file from https://nextcloud.com/install

Upload the .zip file to the public_html folder of your subdomain.

Extract the contents directly into the root of public_html.
锔 Make sure the files are not inside a subfolder after extraction.

![Extracci贸n en SiteGround](https://i.ibb.co/Xr1KQq5y/three.webp)  

##4. Create a PostgreSQL Database
Go to Site Tools > Databases > PostgreSQL.

Create a new database (SiteGround will assign a hash-like name).

Create a user (also hash-style) and set a password.

Assign the user to the database.
 Important: Save the database name, username, and password.


![Creaci贸n de BD PostgreSQL](https://i.ibb.co/hxfMLRtB/four.webp)  
![Asignaci贸n de usuario](https://i.ibb.co/01J6q45/five.webp)
